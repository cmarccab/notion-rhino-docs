在 Notion Rhino 專案管理看板中，**專案就是史詩**，兩者是同一個概念。每個專案頁面就代表一個史詩——頁面標題是它的名稱，頁面內部的內容則定義了它的所有細節。

這種設計讓工作空間保持簡潔扁平。不需要另外維護一個史詩資料庫，所有細節都在專案頁面裡。

## 怎麼運作的

### 專案標題 = 史詩名稱

新建專案時，頁面標題就是這個史詩的名字。建議寫得清楚一點，聚焦在要完成的結果上。

範例：
- `使用者研究與需求探索`
- `設計與原型製作`
- `Q3 行銷活動`
- `Acme 客戶導入`

### 專案頁面內部 = 史詩詳情

開啟一個專案頁面，就能看到這個史詩的完整內容。內部內容用**分頁**來組織，每個分頁對應工作的一個面向：

| 分頁 | 內容 |
|------|------|
| **概覽** | 這個史詩的目標、背景與成功標準 |
| **任務** | 組成這項工作的具體任務與子任務 |
| **時間軸** | 排程與里程碑 |
| **備註** | 會議記錄、決策與進度更新 |

這種分頁架構把所有相關內容集中在一個地方，同時不會讓主看板顯得雜亂。

## 實際層級結構

```
看板（頂層檢視）
└── 專案頁面 = 史詩
    ├── 分頁：概覽
    ├── 分頁：任務
    ├── 分頁：時間軸
    └── 分頁：備註
```

在看板上，你可以一眼看到所有史詩。點進任何一個就能深入處理。

<Tip>
  為專案命名時，要像寫交付成果一樣寫，不要寫成模糊的分類。「設計首頁」比「設計工作」好——前者一眼就知道什麼時候算完成了。
</Tip>
